## Prerequisites

<p data-lang="en">All builds need: a working <code>git</code>, a Rust toolchain (<code>rustup</code>), and the platform's native build tools listed below. The repo ships a <code>script/bootstrap</code> entrypoint that installs the right set for your OS.</p>
<p data-lang="zh">所有平台均需准备:可用的 <code>git</code>、Rust 工具链(<code>rustup</code>),以及下文列出的本机构建工具。仓库提供 <code>script/bootstrap</code> 入口脚本,会根据当前操作系统自动安装对应依赖。</p>
<p data-lang="ja">すべてのビルドには、動作する <code>git</code>、Rust ツールチェーン (<code>rustup</code>)、そして下に示す各プラットフォームのネイティブビルドツールが必要です。リポジトリには OS に合った依存関係を入れる <code>script/bootstrap</code> エントリポイントが含まれています。</p>

## 2. Install build dependencies

<p data-lang="en">Pick your platform. The bootstrap script delegates to <code>script/macos/bootstrap</code>, <code>script/linux/bootstrap</code>, or <code>script/windows/bootstrap.ps1</code> automatically.</p>
<p data-lang="zh">请根据当前平台选择对应步骤。<code>script/bootstrap</code> 会根据操作系统自动调用 <code>script/macos/bootstrap</code>、<code>script/linux/bootstrap</code> 或 <code>script/windows/bootstrap.ps1</code>。</p>
<p data-lang="ja">使っているプラットフォームを選んでください。bootstrap スクリプトは OS に応じて <code>script/macos/bootstrap</code>、<code>script/linux/bootstrap</code>、または <code>script/windows/bootstrap.ps1</code> を自動的に呼び出します。</p>

## 3. Build & run

<p data-lang="en">OpenWarp ships two binaries: <code>warp-oss</code> (no cloud-bound features compiled in — the right target for community builds) and <code>warp</code> (Warp Inc.'s internal channel, requires their private channel config). Use <code>warp-oss</code>.</p>
<p data-lang="zh">OpenWarp 提供两个二进制文件:<code>warp-oss</code>(已移除全部云端能力,社区构建请使用此版本)与 <code>warp</code>(Warp Inc. 内部 channel,依赖其私有 channel 配置)。请使用 <code>warp-oss</code>。</p>
<p data-lang="ja">OpenWarp には 2 つのバイナリがあります。<code>warp-oss</code> はクラウド依存機能を含まないコミュニティビルド向けのターゲットで、<code>warp</code> は Warp Inc. の内部チャンネル用で非公開のチャンネル設定が必要です。<code>warp-oss</code> を使ってください。</p>

## 4. Configure a provider

<p data-lang="en">After launch, open <code>Settings → Agents → Providers</code> and add at least one OpenAI-compatible endpoint. Credentials are written to <code>settings.toml</code> on disk and never sent anywhere except the endpoint you configure.</p>
<p data-lang="zh">启动后,打开 <code>设置 → Agents → 服务商</code>,至少添加一个 OpenAI 兼容端点。凭证仅写入本地 <code>settings.toml</code> 文件,除用户配置的端点外不会发送至任何位置。</p>
<p data-lang="ja">起動後、<code>Settings → Agents → Providers</code> を開き、少なくとも 1 つの OpenAI 互換エンドポイントを追加してください。認証情報はディスク上の <code>settings.toml</code> に書き込まれ、設定したエンドポイント以外へ送られることはありません。</p>

<CodeCard
  file="~/.config/openwarp/settings.toml"
  lang="TOML"
  html={`<span class="key">[providers.deepseek]</span>
base_url = <span class="str">"https://api.deepseek.com/v1"</span>
protocol = <span class="str">"deepseek"</span>     <span class="com"># or: openai, anthropic, gemini, ollama</span>
api_key  = <span class="str">"sk-•••"</span>          <span class="com"># stored locally, never sent home</span>

<span class="key">[[providers.deepseek.models]]</span>
display = <span class="str">"DeepSeek V4 Pro"</span>
id      = <span class="str">"deepseek-v4-pro"</span>
context = <span class="num">1000000</span>
output  = <span class="num">384000</span>`}
/>

<p class="muted" data-lang="en">Hit <code>⌘P</code> to switch model, <code>/</code> to open the slash menu. Cloud Agent, Computer Use, and telemetry ship disabled — see the GitHub repo for the full settings reference.</p>
<p class="muted" data-lang="zh">按 <code>⌘P</code> 切换模型,<code>/</code> 打开 slash 菜单。Cloud Agent、Computer Use 与遥测均默认关闭,完整配置说明请参见 GitHub 仓库。</p>
<p class="muted" data-lang="ja"><code>⌘P</code> でモデルを切り替え、<code>/</code> で slash メニューを開きます。Cloud Agent、Computer Use、テレメトリーは既定で無効です。完全な設定リファレンスは GitHub リポジトリを参照してください。</p>
